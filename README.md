***ONLINE BOOKSTORE MANAGEMENT SYSTEM***
The Online Bookstore Management System is a MySQL database project developed to manage books, authors, genres, publishers, customers, orders, payments, and order details efficiently. This project helps in storing and organizing bookstore information such as book records, customer orders, payment details, and stock management. The database is designed using relational tables with Primary Keys and Foreign Keys to maintain proper relationships between entities.

-->FEATURES:-
Manage author and book information
Store customer details
Track customer orders
Maintain payment records
Manage stock quantity
Generate reports using SQL queries
Perform operations using joins, aggregate functions, and subqueries

-->MODULES:-
1.BOOK MANAGEMENT MODULE
This module manages all book-related information including:
Book title,Price,Stock quantity,Genre details,Publisher details
It helps in maintaining bookstore inventory efficiently.
2.AUTHOR MANAGEMENT MODULE
This module stores author information such as:
Author name,Author biography,Books written by authors
One author can write multiple books.
3.CUSTOMER MANAGEMENT MODULE
This module maintains customer records including:
Customer name,Email,City,Join date
It helps in tracking customer purchase activities.
4.ORDER MANAGEMENT MODULE
This module handles customer orders and order details:
Order date,Ordered books,Quantity ordered,Purchase records,One customer can place multiple orders.
5.PAYMENT MANAGEMENT MODULE
This module stores payment information:
Payment method,Payment amount,Payment date
It helps in managing transaction records for orders.

**BLOOD BANK MANAGEMENT SYSTEM**
The Blood Bank Management System is a MySQL database project developed to manage donors, patients, blood requests, blood stock, donations, staff, and blood issue details efficiently. This project helps in storing and organizing blood bank information such as donor records, patient requests, blood availability, and issued blood details. The database is designed using relational tables with Primary Keys and Foreign Keys to maintain proper relationships between entities.

-->FEATURES:-
Manage donor information
Store patient records
Track blood donations
Maintain blood stock availability
Manage blood requests and issues
Track staff details
Generate reports using SQL queries
Perform operations using joins, aggregate functions, and subqueries

-->MODULES:-
1.DONOR MANAGEMENT
Stores donor details such as:
Name,Blood Group,City,Contact Information,Donation History
2.PATIENT MANAGEMENT
Maintains patient records and blood requirements.
3.BLOOD STOCK MANAGEMENT
Tracks available blood units for different blood groups.
4. REQUEST MANAGEMENT
Handles blood requests raised by patients.
5.BLOOD ISSUE MANAGEMENT
Records issued blood details and request fulfillment.
6.STAFF MANAGEMENT
Maintains staff information and operational handling.
